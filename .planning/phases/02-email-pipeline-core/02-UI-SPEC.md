# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Email Pipeline Core.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Note

Phase 2 UI scope is the **first-boot onboarding wizard** and a **minimal dashboard shell**. The full approval queue UI is Phase 4. Phase 2 delivers the backend API contract (REST + WebSocket) that Phase 4 consumes, and the onboarding surfaces that gate live email processing.

**Phase 2 screens:**
1. Onboarding wizard — step 1: Admin account creation (ONBR-01)
2. Onboarding wizard — step 2: Email connection (OAuth2 redirect or manual IMAP/SMTP) (ONBR-02)
3. Ingest progress screen — 6-month history ingestion with live progress (ONBR-03)
4. Persona tuning screen — 20 sample draft cards, tone rating (ONBR-05, PERS-02)
5. Document upload screen — PDF/DOCX/CSV knowledge base upload (ONBR-04)
6. Notification preferences screen (ONBR-06)
7. Live shell — minimal post-onboarding placeholder directing operator to check back (Phase 4 builds the real queue view)

**Out of scope for Phase 2 UI:** Approval queue actions (approve/edit/reject/escalate), sent history log, classification log, knowledge base management UI, system status view, auto-send controls. These are Phase 4.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding |
| sm | 8px | Inline field padding, compact list item spacing |
| md | 16px | Default form field spacing, card body padding |
| lg | 24px | Section gaps within a wizard step |
| xl | 32px | Between wizard step heading and form body |
| 2xl | 48px | Wizard card vertical margin from viewport edge |
| 3xl | 64px | Not used in Phase 2 surfaces |

**Exceptions:**
- Tone rating buttons (persona tuning screen) use a minimum touch target of 44px height to meet WCAG 2.5.5 on mobile browsers per DASH-09 mobile-responsive requirement.
- Progress bar track height: 8px (not on scale — fixed visual affordance).

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Form labels, email body preview text in persona tuning cards |
| Label | 14px | 400 (regular) | 1.4 | Input field labels, helper text, metadata (confidence score, source label) |
| Heading | 20px | 600 (semibold) | 1.2 | Wizard step titles, screen section headers |
| Display | 28px | 600 (semibold) | 1.2 | Wizard card page title ("Welcome to MailBox One", "Connect Your Email") |

**Rules:**
- Maximum 2 weights in use: 400 and 600.
- Do not use italic in wizard flows — it reads as error state on constrained screens.
- Truncate long email subjects at 1 line with `text-ellipsis overflow-hidden whitespace-nowrap` in persona tuning cards.

## Color

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#FFFFFF` | Page background, wizard card face, form field backgrounds |
| Secondary (30%) | `#F5F5F5` | Wizard outer shell/backdrop, ingest progress container, persona card background before interaction |
| Accent (10%) | `#1D4ED8` | Primary CTA buttons, active step indicator dot, progress bar fill |
| Destructive | `#DC2626` | Reject action button (persona tuning "Wrong Tone" is NOT destructive — see below), inline field validation errors |

**Accent reserved for:**
- Primary CTA button background (`Continue`, `Connect Email`, `Send Test`, `Start Live Email`)
- Current wizard step dot in step indicator
- Progress bar fill during history ingest

**Accent NOT used for:**
- Hover states on secondary actions (use `#E5E7EB` background instead)
- The "Wrong Tone" rating in persona tuning (use `#FEF2F2` background with `#991B1B` text — a warning state, not a destructive action, since it drives training not deletion)
- Borders, shadows, or decorative elements

**Semantic additions:**
- Success: `#16A34A` — used only on "Ingest complete" and "Persona tuning complete" status badges
- Warning: `#D97706` — used only on `awaiting_cloud` queue status badge (visible in the minimal live shell status row)

## Interaction Contracts

### Wizard Step Indicator

- Display: horizontal stepper, 6 dots, numbered 1–6
- Active step: filled accent dot (`#1D4ED8`), 10px diameter
- Completed step: checkmark icon in accent circle, 10px diameter
- Upcoming step: hollow dot, `#D1D5DB` border, 10px diameter
- No clickable back-navigation — steps are linear and gated (operator cannot return to a completed step mid-wizard; they must finish the current screen)

### Ingest Progress Screen

- Progress bar: full-width, 8px height, `#1D4ED8` fill on `#E5E7EB` track
- Label below bar: "{N} of {total} emails imported" — updated via WebSocket push from backend
- Status text: "Importing your sent history — this takes 2–5 minutes for most accounts."
- No spinner — progress bar is the only loading indicator
- On complete: bar fills, label changes to "Import complete — {N} emails indexed", success badge `#16A34A` appears, "Continue" CTA activates

### Persona Tuning Cards

- 20 cards displayed as a vertical scroll list (not a carousel — operator must review all 20)
- Each card: inbound email excerpt (top, `#F5F5F5` background, 3-line max), draft reply preview (bottom, white background, 4-line max), three rating buttons in a row at the bottom
- Card border: `1px solid #E5E7EB`; rated card border changes to color of the chosen rating (green / red / yellow)
- "Confirm My Voice" CTA is disabled until all 20 cards have a rating — show count: "12 of 20 rated" below the list

### Awaiting Cloud Status (minimal live shell)

- After onboarding completes, the minimal live shell shows a status row: "Email processing is live" badge
- If any queue items have `status = 'awaiting_cloud'`: show a non-blocking banner — "Some drafts are waiting for cloud processing. They'll appear when your connection is restored." — `#FEF9C3` background, dismissable with an X, re-appears on next page load if items still exist

## Accessibility

- All form inputs: `<label for>` association required — no placeholder-only labels
- Tone rating buttons: `aria-pressed` state on selected option per card
- Wizard step indicator: `aria-current="step"` on active dot
- Progress bar: `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`
- Touch targets: minimum 44px height on all interactive elements (DASH-09 mobile-first)
- Color contrast: all text on accent (`#1D4ED8`) backgrounds uses white (`#FFFFFF`) — contrast ratio 4.5:1 minimum
