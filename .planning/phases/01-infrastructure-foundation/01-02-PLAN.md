Create `scripts/first-boot.sh` as a bash script with `#!/usr/bin/env bash` and `set -euo pipefail`.

**Script structure:** Implement a `run_stage` function per D-01/D-02 — each stage retries once on failure, then halts with diagnostics. Between each stage, pause and prompt the user to press Enter to continue (per D-01 "manual verification between each stage").

**Script helper functions:**

`run_stage()` — takes stage name and function name. Calls function. On failure, retries once after 5s delay. On second failure, prints `[FAIL]` with stage name and exits 1. On success, prints `[PASS]`.

`pause_for_verification()` — prints "Press Enter to continue to next stage..." and reads input. Allows operator to inspect output before proceeding.

**Stage 1: Validate JetPack Version (per D-03)**

IMPORTANT: This script does NOT flash the device. JetPack 6.2 must already be installed via NVIDIA SDK Manager before running this script. Stage 1 validates the pre-existing installation and aborts with remediation guidance if the version is wrong.

- Read `/etc/nv_tegra_release`
- If file does not exist: print `"ERROR: /etc/nv_tegra_release not found. This device does not appear to have JetPack installed. Flash the device with JetPack 6.2.2 using NVIDIA SDK Manager (https://developer.nvidia.com/sdk-manager) before running this script."` and exit 1.
- Extract version string matching `R[0-9]+` pattern (e.g., `R36`)
- Also extract revision from line (e.g., `REVISION: 5.0`)
- Print full version info
- Fail if major version < R36 with message: `"ERROR: JetPack version too old. Need R36 (revision 4.0+) = JetPack 6.2+. Re-flash with NVIDIA SDK Manager using JetPack 6.2.2."`
- Warn (but continue) if revision < 5.0: `"WARNING: r36.4 detected. r36.5 (JetPack 6.2.2) recommended for CUDA memory allocation fix."`
- On retry: re-read file (no fix possible — just re-check in case of read error)

**Stage 2: Install Docker via JetsonHacks (per INFRA-02, STATE.md decision)**
- Check if Docker is already installed: `docker --version 2>/dev/null`
- If installed, print version and skip install
- If not installed:
  - `git clone https://github.com/jetsonhacks/install-docker.git /tmp/install-docker`
  - `cd /tmp/install-docker && bash install_nvidia_docker.sh`
  - `bash configure_nvidia_docker.sh`
- Verify: `docker --version` must succeed
- On retry: re-run install scripts
- Diagnostics on fail: print `docker --version` output, suggest checking internet connection and JetsonHacks repo

**Stage 3: Verify GPU Passthrough (per INFRA-02)**
- Run: `docker run --rm --runtime nvidia nvidia/cuda:12.3.0-base-ubuntu22.04 nvidia-smi`
- Capture exit code; if non-zero, fail
- Print nvidia-smi output
- On retry: restart docker daemon (`sudo systemctl restart docker`), then retry
- Diagnostics on fail: print `nvidia-ctk --version`, suggest running `nvidia-ctk runtime configure --runtime=docker && sudo systemctl restart docker`

**Stage 4: Set MAXN Power Mode (per INFRA-03)**
- Run `sudo nvpmodel -q --verbose` and capture output
- Parse output to find mode with "MAXN" in the name (case-insensitive grep for "MAXN")
- Extract the mode ID number
- Run `sudo nvpmodel -m <detected_mode_id>`
- Verify: `sudo nvpmodel -q` output contains "MAXN" (case-insensitive)
- Create systemd service `/etc/systemd/system/set-maxn-power.service`:
  ```
  [Unit]
  Description=Set Jetson Orin to MAXN power mode
  After=multi-user.target

  [Service]
  Type=oneshot
  ExecStart=/usr/bin/nvpmodel -m <detected_mode_id>
  RemainAfterExit=yes

  [Install]
  WantedBy=multi-user.target
  ```
- Enable service: `sudo systemctl enable set-maxn-power.service`
- On retry: re-query modes
- Diagnostics on fail: print all available modes from `nvpmodel -q --verbose`

**Stage 5: LUKS Encrypt Data Partition (per D-09, D-10, INFRA-11)**
- Install prerequisites: `sudo apt-get update && sudo apt-get install -y cryptsetup-bin tpm2-tools`
- Check if `/dev/tpm0` or `/dev/tpmrm0` exists; warn if not found
- Check if `/usr/sbin/gen_luks.sh` exists
- If gen_luks.sh exists:
  - Print: `"Using Jetson-native LUKS encryption (gen_luks.sh + OP-TEE luks-srv)"`
  - Identify the data partition (prompt user to confirm partition, e.g., `/dev/nvme0n1p4`)
  - Run: `sudo /usr/sbin/gen_luks.sh <partition>`
  - Verify: `sudo cryptsetup luksDump <partition>` shows LUKS header
- If gen_luks.sh does NOT exist:
  - Print: `"WARNING: gen_luks.sh not found. Attempting to install nvidia-l4t-security-utils..."`
  - Try: `sudo apt-get install -y nvidia-l4t-security-utils`
  - Re-check for gen_luks.sh
  - If still not found: print `"FAIL: LUKS encryption requires gen_luks.sh. See NVIDIA Jetson Linux r36.5 Developer Guide > Disk Encryption."` and exit 1
- On retry: re-check for gen_luks.sh after package install
- Diagnostics: print `ls /usr/sbin/gen_luks*`, `dpkg -l | grep nvidia-l4t-security`

**Stage 6: Pre-pull Ollama Models (per INFRA-06, INFRA-07)**
- Source `.env` from repo root to get OLLAMA_IMAGE (default: `dustynv/ollama:0.18.4-r36.4-cu126-22.04`)
- Check if `jetson-containers` is available; if so, resolve image via `autotag ollama` and print result
- Create the named volume if it does not exist: `docker volume create ollama_models`
- Pull Qwen3-4B: `docker run --rm --runtime nvidia -v ollama_models:/root/.ollama ${OLLAMA_IMAGE} ollama pull qwen3:4b`
- Pull nomic-embed-text: `docker run --rm --runtime nvidia -v ollama_models:/root/.ollama ${OLLAMA_IMAGE} ollama pull nomic-embed-text:v1.5`
- Verify models exist: `docker run --rm -v ollama_models:/root/.ollama ${OLLAMA_IMAGE} ollama list` should show both models
- On retry: re-pull (network issue)
- Diagnostics: print disk space (`df -h`), docker images list

**Stage 7: Start Docker Compose Stack**
- `cd` to the repo root (script should detect its own location: `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`, repo root is `$SCRIPT_DIR/..`)
- Check `.env` file exists; if not, copy from `.env.example` and warn user to edit passwords
- Run: `docker compose up -d`
- Wait for all services to be healthy: loop checking `docker compose ps --format json` for health status, timeout after 180 seconds (3 minutes per INFRA-12)
- Print final status: `docker compose ps`
- On retry: `docker compose down && docker compose up -d`
- Diagnostics: `docker compose logs --tail=20`

**Final output:**
- Print summary table showing each stage status (PASS/SKIP)
- Print: `"First-boot complete. Run scripts/smoke-test.sh to verify all services."`

**Script must:**
- Run as root or with sudo (check at start: `if [[ $EUID -ne 0 ]]; then echo "Run with sudo"; exit 1; fi`)
- Be executable (`chmod +x`)
- Use absolute paths where possible
- Handle Ctrl+C gracefully (trap)
