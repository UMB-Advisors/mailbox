  <action>
Create `scripts/smoke-test.sh` at repo root (per D-13) with `#!/usr/bin/env bash` and `set -euo pipefail`.

**Script structure:**

Define color constants (GREEN, RED, YELLOW, NC) for terminal output. Define counters: PASS=0, FAIL=0, TOTAL=6. Define a `run_check` function that takes a check name and function name, runs it, captures exit code, increments PASS or FAIL, prints colored PASS/FAIL result. Trap on EXIT to always print the summary.

**Check 1: GPU Passthrough (maps to Success Criterion 2)**
- Run: `docker run --rm --runtime nvidia nvidia/cuda:12.3.0-base-ubuntu22.04 nvidia-smi`
- Capture output into variable
- Assert exit code is 0
- Assert output contains "NVIDIA" (confirms GPU is detected)
- Print GPU name from nvidia-smi output
- On failure: print diagnostic — `nvidia-ctk --version 2>/dev/null || echo "nvidia-container-toolkit not found"`

**Check 2: Qwen3-4B Inference < 5 seconds (maps to Success Criterion 3, INFRA-06)**
- Record start time: `START=$(date +%s%N)`
- Run inference via Ollama API:
  ```
  curl -s --max-time 30 http://localhost:11434/api/generate \
    -d '{"model":"qwen3:4b","prompt":"Reply with exactly: hello world","stream":false,"options":{"num_predict":20}}'
  ```
- Record end time, compute duration in seconds (using nanosecond math: `$(( (END - START) / 1000000000 ))` with millisecond remainder)
- Assert exit code is 0
- Assert response JSON contains `"response"` field (not empty)
- Assert duration < 5 seconds (5000ms)
- Print: model name, inference time, first 50 chars of response
- Also verify GPU layers: run `curl -s http://localhost:11434/api/show -d '{"model":"qwen3:4b"}'` and check that response contains a non-zero value — print the num_gpu value found
- On failure: print `curl -s http://localhost:11434/api/tags` to show loaded models

**Check 3: nomic-embed-text Embeddings (maps to Success Criterion 3, INFRA-07)**
- Run embedding via Ollama API:
  ```
  curl -s --max-time 30 http://localhost:11434/api/embed \
    -d '{"model":"nomic-embed-text:v1.5","input":"test embedding for smoke check"}'
  ```
- Assert exit code is 0
- Assert response JSON contains `"embeddings"` field
- Assert embeddings array is non-empty (length > 0) — use a simple check: response contains `[` after `"embeddings":`
- Print: embedding dimension count (parse JSON to count comma-separated values in first embedding vector, or just confirm non-empty)
- On failure: print `curl -s http://localhost:11434/api/tags`

**Check 4: Qdrant Health — No jemalloc Errors (maps to Success Criterion 4, INFRA-08)**
- Run: `curl -s --max-time 10 http://localhost:6333/healthz`
- Assert response is `ok` or HTTP 200
- Check Qdrant container logs for jemalloc errors: `docker compose logs qdrant 2>&1 | grep -i "jemalloc\|alloc.*error\|SIGKILL\|OOM" | head -5`
- If any matches found: FAIL with message "Qdrant has jemalloc/memory errors — check MALLOC_CONF setting"
- If no matches: PASS
- Print: Qdrant version from `curl -s http://localhost:6333/ | grep -o '"version":"[^"]*"'` (or similar JSON parse)
- On failure: print `docker compose logs qdrant --tail=30`

**Check 5: Postgres Persistence Across Restart (maps to Success Criterion 4, INFRA-09)**
- Insert a test row: `docker compose exec -T postgres psql -U ${POSTGRES_USER:-mailbox} -d ${POSTGRES_DB:-mailbox} -c "CREATE TABLE IF NOT EXISTS mailbox.smoke_test (id serial, val text); INSERT INTO mailbox.smoke_test (val) VALUES ('smoke-$(date +%s)');"` 
- Capture the inserted value
- Restart postgres: `docker compose restart postgres`
- Wait for postgres to be healthy: loop up to 30 seconds checking `docker compose exec -T postgres pg_isready -U ${POSTGRES_USER:-mailbox}` 
- Query the test row: `docker compose exec -T postgres psql -U ${POSTGRES_USER:-mailbox} -d ${POSTGRES_DB:-mailbox} -c "SELECT val FROM mailbox.smoke_test ORDER BY id DESC LIMIT 1;" -t`
- Assert the returned value matches what was inserted
- Cleanup: `docker compose exec -T postgres psql -U ${POSTGRES_USER:-mailbox} -d ${POSTGRES_DB:-mailbox} -c "DROP TABLE IF EXISTS mailbox.smoke_test;"`
- On failure: print `docker compose logs postgres --tail=20`

**Check 6: Boot Time < 3 Minutes (maps to INFRA-05, INFRA-12)**
- Bring the stack down: `docker compose down`
- Record start time: `BOOT_START=$(date +%s)`
- Bring stack up: `docker compose up -d`
- Poll for all services healthy: loop checking `docker compose ps --format '{{.Health}}'` (or `docker compose ps` and grep for "healthy"), timeout after 240 seconds
- Record end time when ALL services report healthy: `BOOT_END=$(date +%s)`
- Compute duration: `BOOT_SECONDS=$((BOOT_END - BOOT_START))`
- Assert `BOOT_SECONDS < 180` (3 minutes)
- Print: "Boot time: ${BOOT_SECONDS}s (limit: 180s)"
- On failure: print which services are not healthy — `docker compose ps`, `docker compose logs --tail=10`

**Summary output (printed by EXIT trap):**
```
===================================
SMOKE TEST RESULTS — $(hostname)
Date: $(date -u +"%Y-%m-%dT%H:%M:%SZ")
===================================
GPU Passthrough:     PASS/FAIL
Qwen3-4B Inference:  PASS/FAIL  (Xs)
Embeddings:          PASS/FAIL
Qdrant Health:       PASS/FAIL
Postgres Persist:    PASS/FAIL
Boot Time:           PASS/FAIL  (Xs)
-----------------------------------
Total: X/6 passed
===================================
```

Store each check result in an associative array `declare -A RESULTS` keyed by check name, with value "PASS" or "FAIL". The EXIT trap iterates and prints the table.

Exit code: 0 if all 6 pass, 1 if any fail.

**Script must:**
- Source `.env` from repo root if it exists (for POSTGRES_USER, POSTGRES_DB values)
- Accept `--skip-boot-test` flag to skip Check 6 (useful when running just the functional checks without tearing down the stack)
- Be executable (`chmod +x`)
- Print elapsed time for the entire smoke test run
- Use `#!/usr/bin/env bash` shebang
