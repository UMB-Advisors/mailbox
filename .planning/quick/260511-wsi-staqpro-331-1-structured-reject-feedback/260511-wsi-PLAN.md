<objective>
Land structured reject feedback for STAQPRO-331 item #1: replace the freeform `reason` string write on `mailbox.drafts.error_message` with a dedicated append-only `mailbox.draft_feedback` table + 6-code taxonomy, port the sandbox reject popover to the production dashboard, and consolidate the reason-code SoT in `lib/types.ts` (mirrored against the Postgres CHECK constraint by schema-invariants).

Purpose: This is the on-ramp for the learning loop in items #2–#8 of STAQPRO-331 (persona signal from `wrong_tone`, RAG signal from `factually_inaccurate`/`missing_context`, classifier signal from `dont_reply`). Today's reject collapses every rejection into one freeform string on the wrong column (`drafts.error_message` is the send-failure column per state-machine doc). Without structured feedback there is nothing to feed those downstream consumers, and the rejected-history table is opaque for retrospective analysis.

      ```ts
      import { REJECT_REASON_CODES, type RejectReasonCode } from '@/lib/types';

      // POST /api/drafts/[id]/reject — accepts BOTH shapes during a deprecation window:
      //   NEW (STAQPRO-331 #1): { reason_code: RejectReasonCode, free_text?: string | null }
      //                          — `other` requires non-empty free_text (zod .refine())
      //   LEGACY (deprecated):    { reason?: string } — mapped to
      //                          { reason_code: 'other', free_text: reason }
      // TODO(STAQPRO-331 follow-up): remove the legacy alias one release after this lands
      // (search call sites: rg "/api/drafts/.+/reject" — no callers should be sending
      // `{ reason }` after the QueueClient port in this same patch).
      const reasonCodeEnum = z.enum(REJECT_REASON_CODES as readonly [RejectReasonCode, ...RejectReasonCode[]]);

      const newRejectShape = z.object({
        reason_code: reasonCodeEnum,
        free_text: z.string().trim().min(1).max(2000).optional().nullable().transform((v) => v ?? null),
      }).refine(
        (v) => v.reason_code !== 'other' || (v.free_text !== null && v.free_text.length > 0),
        { message: "free_text is required when reason_code is 'other'", path: ['free_text'] },
      );

      const legacyRejectShape = z.object({
        reason: z.string().trim().min(1).max(2000),
      }).transform((v) => ({ reason_code: 'other' as const, free_text: v.reason }));

      // Accept empty body as well (back-compat with the current handler's `parseJson` empty-body path).
      const emptyBody = z.object({}).strict().transform(() => null);

      export const rejectBodySchema = z.union([newRejectShape, legacyRejectShape, emptyBody]);
      export type RejectBody = z.infer<typeof rejectBodySchema>;
      ```

The `emptyBody` branch parses to `null` and the route handler returns 400 when `parsed === null`. This preserves the test invariant that already-sent drafts with empty body still return 409 first (status guard runs before content guard) — see route logic below.

      ```ts
      if (!p.ok) return p.response;

      const b = await parseJson(req, rejectBodySchema);
      if (!b.ok) return b.response;
      ```
